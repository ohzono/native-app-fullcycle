# 機能実装計画

新機能「$ARGUMENTS」の実装計画を作成します。

## 引数について

- **指定あり**: 指定された機能について計画を作成
- **指定なし**: 対話的に機能要件をヒアリング

## 実行フロー

1. **コンテキスト収集**: プロジェクト構造、技術スタック、既存パターンを分析
2. **仕様の独立検証**: 設計案を「正」と鵜呑みにせず、代替案を検討
3. **要件の明確化**: 機能要件、非機能要件、制約条件、成功基準を整理
4. **技術調査**: 必要に応じてベストプラクティスを調査
5. **計画出力**: Phase分割、タスクリスト、テスト戦略、リスク評価

計画の詳細な策定と出力は `feature-planner` エージェントの定義に従ってください。

## 実行例

```
/dev-plan ユーザー認証機能
/dev-plan ダッシュボード画面の追加
/dev-plan  # 対話的にヒアリング
```

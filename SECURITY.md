# セキュリティポリシー

`native-app-fullcycle` の脆弱性報告窓口と対応方針を記載します。安全に脆弱性を報告いただくため、ご一読ください。

## サポート対象バージョン

本プラグインは marketplace から HEAD（最新の `main`）を配布する運用です。そのため、セキュリティ対応の対象は **最新リリース（最新 `main` 相当）のみ** です。

| バージョン | サポート |
|-----------|---------|
| 最新リリース（最新 `main` 相当） | サポート対象 |
| それ以前の任意のコミット | サポート対象外（最新へ更新してください） |

脆弱性の影響を受ける可能性がある場合は、まず最新版へ更新してください。

## 脆弱性の報告

**脆弱性は公開 Issue / PR に投稿しないでください。** 公開の場で詳細を共有すると、修正前に悪用される恐れがあります。

報告は GitHub の **private vulnerability reporting** を使って非公開でお願いします。

1. 本リポジトリの **Security** タブを開く
2. **Advisories** → **Report a vulnerability** を選択
3. 影響範囲・再現手順・想定されるリスクをできるだけ具体的に記載して送信

この経路は GitHub Security Advisories の非公開チャネルで、メンテナのみが内容を閲覧できます。

## 対応方針

対応は **ベストエフォート** で行います（本プラグインは個人メンテナンスの OSS です）。

- **一次応答の目安**: 受領から **7 日以内** に受領確認を返します。
- **修正**: 深刻度に応じて優先度を判断し、可能な範囲で修正します。
- **公開**: 必要に応じて、修正後に GitHub Security Advisory として内容を公開する場合があります。報告者のクレジットは希望を確認のうえ記載します。

応答目安はあくまで目標であり、保証する SLA ではありません。

## 対象範囲

本リポジトリが配布する以下の資産を対象とします。

- `skills/` 配下のスキル
- `agents/` 配下のエージェント
- `commands/` 配下のコマンド・フェーズ定義
- `scripts/` 配下のスクリプト
- `.claude/settings.example.json`（および権限設定の例）

### 同梱の第三者資産について

本プラグインには以下の第三者資産が vendored copy として同梱されています。

- [android/skills](https://github.com/android/skills)（`vendor/android-skills/`）
- [SwiftUI-Agent-Skill](https://github.com/twostraws/SwiftUI-Agent-Skill)（`skills/swiftui-pro/`）
- Gradle wrapper（再現確認用プロジェクトに含まれる場合）

これら **第三者資産そのものの脆弱性は、各 upstream リポジトリへ報告** してください。ただし、本プラグインでの **取り込み方法**（同期スクリプト、pin した SHA、改変の有無など）に起因する問題は、本リポジトリへ報告してください。
